# PetVerse 全游戏 UI 整改任务书 V1.0

## 0. 文档定位

本任务书用于整改仓库 `BGHOLY/petverse` 的 Cocos Creator 客户端 UI。  
目标不是简单换色，而是在保留现有业务功能和接口的前提下，把当前“程序动态绘制、网页卡片式”的界面，升级成统一的奶油手绘、纸张、木质、温馨宠物养成游戏风格。

### 当前基线

- 仓库：`BGHOLY/petverse`
- 分支：`main`
- 本次审查基线提交：`c6a69565a6fbf0f62a8325b9c20866278cf86589`
- Cocos 客户端：`client/PetVerseClient`
- 设计分辨率：`720 × 1280`
- 当前主要 UI 入口：`client/PetVerseClient/assets/scripts/ui/MainUI.ts`
- 当前公共绘制工具：`client/PetVerseClient/assets/scripts/ui/cute/CuteUiKit.ts`

### 不得擅自修改

除非对应阶段明确要求，否则禁止修改：

- 后端业务规则
- 数据库结构
- `.env`
- MySQL 配置
- 登录鉴权
- 现有接口路径
- 宠物、背包、商城、婚姻、孵化、技能、炼妖、交易、邮件等已经可用的业务逻辑

需要新增接口时，先通过适配器与 Mock 数据完成 UI，再单独列出接口缺口，不得用假按钮冒充已实现功能。

# 2. 当前代码审查结论

## 2.1 当前优点

以下能力必须保留：

- 已固定 `720 × 1280` 设计分辨率
- 已有顶部栏、页面根节点、底部栏、弹窗层、战斗层、Toast 层
- 已有页面切换和返回历史
- 已有按钮点击反馈、页面动画、声音反馈
- 已有真实后端数据接入
- 已有宠物立绘注册和自动绑定
- 已有背包使用对象、商店购买、孵化、打书、炼妖、编队、战斗回放等逻辑
- 页面切换时已避免部分旧页面叠加问题

## 2.2 当前主要问题

### A. `MainUI.ts` 职责过重

当前首页、宠物、背包、商店、孵化、技能、炼妖、冒险、好友、婚姻、邮件、排行、交易、设置等几乎全部集中在同一个文件中。

后果：

- 修改一个页面容易影响其他页面
- 页面状态容易被重新渲染清空
- 难以复用统一组件
- 难以在 Cocos 编辑器中直观看到静态层级
- 后续接入正式 PNG 资源时维护成本极高

### B. 页面主要由运行时 Graphics 动态绘制

当前 `CuteUiKit.ts` 主要使用 `Graphics` 画圆角矩形，并使用 Emoji 充当图标。

最终版本必须改为：

- 透明 PNG
- 九宫格 Sprite
- Sprite Atlas
- 正式图标
- Prefab
- Cocos 编辑器中的可见节点结构

Graphics 可以保留为调试兜底，但不作为正式 UI 主体。

### C. 底部导航不符合最新要求

当前底部为：

```text
家园 / 宝宝 / 背包 / 冒险 / 更多
```

必须改为：

```text
家园 / 宠物 / 冒险 / 商店 / 更多
```

背包移入“更多”。

### D. 当前编队仍是三宠

当前代码存在多处三宠硬编码：

- 只显示三个编队槽
- 只能选择最多三只宠物
- 保存前端只保留前三个 ID
- 战斗回放只截取前三只宠物

必须整体升级为五宠，禁止只改显示数量而遗漏保存、战力计算、战斗和接口数据。

### E. 当前孵化室只有一个孵化槽

必须升级为三个槽位，并支持槽位独立状态、独立倒计时、独立加速和独立领取。

### F. 当前冒险不是主线探索结构

当前主要为：

- 爬塔
- 日常试炼
- 好友切磋

必须升级为拥有主线章节、探索度、区域事件和首领巢穴的冒险大陆，同时保留爬塔和好友玩法入口。

### G. 当前战斗属于三对三回放

当前主要是：

- 暂停
- 倍速
- 查看战报
- 跳过动画

必须升级为五对五阵法战斗，并加入：

- 集火
- 守护
- 套盾
- 净化
- 阵法大招
- 自动战斗

不能只做装饰按钮。如果服务器仍是“一次性结算整场战斗”，需要明确增加回合制战斗会话接口；在接口完成前，按钮必须标注开发状态，不能伪装为已生效。

# 9. 五宠编队与五阵法

## 9.1 五宠编队

每个编队固定最多五只宠物：

```text
1号位 / 2号位 / 3号位 / 4号位 / 5号位
```

所有涉及编队的代码必须统一改为五只：

- 前端状态
- 显示
- 选择
- 排序
- 交换
- 保存
- 战力计算
- 战斗入场
- 战斗回放
- 好友战
- 爬塔
- 主线
- 首领战

禁止残留：

```text
slice(0, 3)
length < 3
已选择 x/3
三宠编队
```

## 9.2 五种阵法

当前确定五种阵法：

- 龙阵
- 龟阵
- 鹤阵
- 虎阵
- 凤阵

阵法名称、说明、加成和大招必须配置化，禁止写死在页面代码中。

### 阵法定义

```ts
interface FormationDefinition {
  id: string;
  name: string;
  icon: string;
  description: string;
  positions: Array<{
    position: 1 | 2 | 3 | 4 | 5;
    x: number;
    y: number;
    bonusType: string;
    bonusValue: number;
    bonusText: string;
  }>;
  counters: string[];
  counteredBy: string[];
  ultimate: {
    name: string;
    description: string;
    energyCost: number;
    icon: string;
  };
}
```

### 编队保存结构

建议最终统一为：

```ts
interface TeamFormationPayload {
  formationId: string;
  slots: Array<{
    position: 1 | 2 | 3 | 4 | 5;
    petId: number;
  }>;
}
```

兼容当前 `/team/set` 时可由 Adapter 临时转换，但最终后端需要保存位置和阵法。

## 9.3 编队页面布局

上方：

- 标题
- 总战力
- 阵法克制提示

阵法页签：

- 龙阵
- 龟阵
- 鹤阵
- 虎阵
- 凤阵

中间：

- 五个阵法站位
- 每个站位显示位置加成
- 宠物卡片落在对应位置
- 点击两个位置可交换
- 选中宠物后点击空位可放入
- 可选支持拖拽，但点击交换必须存在

下方：

- 可用宠物列表
- 已上阵标记
- 待放入标记
- 筛选和排序
- 取消修改

## 9.4 阵法大招

每个阵法拥有独立大招：

- 名称
- 图标
- 能量需求

## 14.2 战斗指令

底部指令：

- 集火
- 守护
- 套盾
- 净化
- 阵法大招
- 自动战斗

### 集火

- 点击后进入目标选择
- 敌方目标高亮
- 再次点击确认
- 本回合或后续若干回合优先攻击该目标

### 守护

- 选择我方宠物
- 指定保护目标
- 显示守护连线或状态图标

### 套盾

- 选择我方目标
- 显示护盾值和持续回合

### 净化

- 选择存在负面状态的我方宠物
- 无可净化目标时禁用

### 阵法大招

- 显示能量条
- 能量不足禁用
- 可释放时高亮和轻动画
- 点击后显示效果预览或确认
- 每个阵法使用不同大招

### 自动战斗

- 开启和关闭
- 开启后按钮显示选中
- 不影响查看战报和倍速

## 14.3 战斗接口要求

当前一次性返回整场结果的接口不足以支持真实战斗内指令。

最终建议接口：

```text
POST /battle/session/start
POST /battle/session/:id/command
POST /battle/session/:id/next-round
GET  /battle/session/:id/state
```

指令结构：

```ts
interface BattleCommand {
  type: 'focus' | 'guard' | 'shield' | 'cleanse' | 'formation_ultimate' | 'auto';
  sourcePetId?: number;
  targetPetId?: number;
  enabled?: boolean;
}
```

UI 不得伪造指令生效结果。

## 14.4 战斗回放

保留：

- 暂停
- 1x / 2x / 4x
- 跳过动画
- 完整战报
- 再战
- 返回冒险

升级：

- 支持五对五
- 支持阵法站位
- 支持阵法大招表现
- 支持集火目标
- 支持状态图标
- 支持回合时间轴
